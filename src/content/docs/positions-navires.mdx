Cette partie est plus avancée et demande une bonne compréhension de la gestion d'état et du flux de données entre composants. Vous allez créer un formulaire interactif permettant de positionner les navires sur une grille.

<Aside type="caution" title="Prérequis — une partie doit exister en base">
Côté backend, le placement des navires se fait **toujours dans le contexte d'une partie** : la table des positions référence à la fois un navire, un joueur et une partie. Avant de pouvoir enregistrer quoi que ce soit depuis le front, il faut donc qu'il existe en base une partie à laquelle votre joueur est rattaché.

Pour démarrer rapidement, **sans construire encore l'écran de matchmaking** :

1. **Créer la partie en base** — via votre client de base de données (ou l'interface d'admin du backend), insérez manuellement une ligne dans la table des parties et **notez son identifiant** (par exemple `partie-1`).
2. **Inscrire votre joueur** — ajoutez votre utilisateur comme participant de cette partie (selon le schéma de votre backend, c'est soit un champ direct, soit une table de liaison).
3. **Mettre l'identifiant en dur côté front** — dans `battleship.service.ts`, déclarez une constante et utilisez-la dans l'URL de l'appel POST de positionnement :

   ```ts
   const PARTIE_KEY = 'partie-1';
   ```

Cette valeur en dur n'est qu'une **étape intermédiaire** pour avancer sur la mécanique du placement. Plus tard (vue *Rejoindre une partie*), l'identifiant viendra de la navigation utilisateur — la constante disparaîtra alors.
</Aside>

## Formulaire d'ajout de navire sur la carte

Revenez sur la page où vous avez listé les navires.

<Task title="Concevoir avant de coder">

Dans cette partie, vous devez **avant de développer** :
- réaliser un schéma pour déterminer les différents composants à écrire ;
- dans chacun des composants : indiquer qui est propriétaire de quel état ;
- indiquer quelle valeur est injectée dans quel composant. Pensez aussi aux callbacks ;
- réfléchir à l'interface de vos composants.

Puis implémenter les points suivants :
- un formulaire devra inclure : la position et l'orientation ;
- soumettre le contenu du formulaire au serveur (pensez à ajouter la `key` du navire).

</Task>

## Ajouter dans le formulaire les valeurs enregistrées

<Task title="Pré-remplir le formulaire avec les positions existantes">

Dans cette partie, vous devez :
- récupérer les positions des navires ;
- ajouter les positions en valeur par défaut dans le formulaire ;
- soumettre le contenu du formulaire au serveur pour créer ou mettre à jour la position du navire ;
- *(facultatif)* ajouter un indicateur sur la liste des navires pour savoir s'il est déjà enregistré.

</Task>

<Aside type="note">
À partir de cette partie, il est recommandé d'utiliser la librairie **Tanstack Query** qui vous permettra de mieux gérer l'asynchronisme des requêtes.

[Documentation Tanstack Query](https://tanstack.com/query/latest/docs/react/overview)
