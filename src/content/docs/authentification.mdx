## Formulaire

<Task title="Créer la page de connexion">

Sur une nouvelle page (par exemple : `/connexion`), créez un formulaire pour permettre l'authentification.
Stockez les valeurs dans une variable d'état.

Il n'y a pas d'attente design sur ce formulaire.

</Task>

### Étape 1 : Formulaire avec un seul champ

<Example title="Étape 1 : Un champ et la gestion de soumission">

Commençons par créer un formulaire simple avec un seul champ pour le nom d'utilisateur :

```tsx title="src/pages/LoginPage.tsx"
import React, { useState } from 'react';
import { Box, Button, Input, Heading, VStack, Field } from '@chakra-ui/react';

const LoginPage: React.FC = () => {
  const [username, setUsername] = useState('');

  const handleSubmit = (e: React.FormEvent) => {
    e.preventDefault();
    console.log('Nom d\'utilisateur:', username);
  };

  return (
    <Box maxW="md" mx="auto" mt={8} p={6} borderWidth="1px" borderRadius="lg">
      <Heading mb={6} textAlign="center">Connexion</Heading>

      <form onSubmit={handleSubmit}>
        <VStack gap={4}>
          <Field.Root>
            <Field.Label>Nom</Field.Label>
            <Input
              type="text"
              value={username}
              onChange={(e) => setUsername(e.target.value)}
            />
          </Field.Root>

          <Button type="submit" width="full" mt={4} variant="outline">
            Se connecter
          </Button>
        </VStack>
      </form>
    </Box>
  );
};

export default LoginPage;
```

Dans cette première étape :
- création d'un état `username` avec `useState`
- champ de saisie qui met à jour cet état via `onChange`
- fonction `handleSubmit` qui affiche la valeur saisie

</Example>

### Étape 2 : Ajout du mot de passe et état de soumission

<Example title="Étape 2 : Mot de passe et état isSubmitting">

Améliorons notre formulaire en ajoutant un champ pour le mot de passe et un état pour gérer le processus de soumission :

```tsx title="src/pages/LoginPage.tsx"
import React, { useState } from 'react';
import { Box, Button, Input, Heading, VStack, Field } from '@chakra-ui/react';

const LoginPage: React.FC = () => {
  const [username, setUsername] = useState('');
  const [password, setPassword] = useState('');
  const [isSubmitting, setIsSubmitting] = useState(false);

  const handleSubmit = (e: React.FormEvent) => {
    e.preventDefault();
    setIsSubmitting(true);
    console.log('Données à envoyer:', { username, password });

    // Simulation d'un appel API
    setTimeout(() => {
      setIsSubmitting(false);
    }, 1000);
  };

  return (
    <Box maxW="md" mx="auto" mt={8} p={6} borderWidth="1px" borderRadius="lg">
      <Heading mb={6} textAlign="center">Connexion</Heading>

      <form onSubmit={handleSubmit}>
        <VStack gap={4}>
          <Field.Root>
            <Field.Label>Nom</Field.Label>
            <Input
              type="text"
              value={username}
              onChange={(e) => setUsername(e.target.value)}
            />
          </Field.Root>

          <Field.Root>
            <Field.Label>Mot de passe</Field.Label>
            <Input
              type="password"
              value={password}
              onChange={(e) => setPassword(e.target.value)}
            />
          </Field.Root>

          <Button
            type="submit"
            width="full"
            mt={4}
            variant="outline"
            loading={isSubmitting}
          >
            Se connecter
          </Button>
        </VStack>
      </form>
    </Box>
  );
};

export default LoginPage;
```

Nouveautés de cette étape :
- état `password` pour le mot de passe
- état `isSubmitting` pour indiquer que le formulaire est en cours de soumission
- simulation de délai pour montrer l'état de chargement du bouton

</Example>

### Étape 3 : Validation des champs

<Example title="Étape 3 : Validation et messages d'erreur">

Ajoutons une validation simple des champs et une gestion des erreurs :

```tsx title="src/pages/LoginPage.tsx"
import React, { useState } from 'react';
import { Box, Button, Input, Heading, VStack, Field } from '@chakra-ui/react';

const LoginPage: React.FC = () => {
  const [username, setUsername] = useState('');
  const [password, setPassword] = useState('');
  const [isSubmitting, setIsSubmitting] = useState(false);

  const isUsernameError = username === '';
  const isPasswordError = password === '';

  const handleSubmit = (e: React.FormEvent) => {
    e.preventDefault();

    if (isUsernameError || isPasswordError) {
      return;
    }

    setIsSubmitting(true);
    console.log('Données à envoyer:', { username, password });

    setTimeout(() => {
      setIsSubmitting(false);
    }, 1000);
  };

  return (
    <Box maxW="md" mx="auto" mt={8} p={6} borderWidth="1px" borderRadius="lg">
      <Heading mb={6} textAlign="center">Connexion</Heading>

      <form onSubmit={handleSubmit}>
        <VStack gap={4}>
          <Field.Root required invalid={isUsernameError}>
            <Field.Label>Nom</Field.Label>
            <Input
              type="text"
              value={username}
              onChange={(e) => setUsername(e.target.value)}
            />
            {isUsernameError && (
              <Field.ErrorText>Le nom est requis</Field.ErrorText>
            )}
          </Field.Root>

          <Field.Root required invalid={isPasswordError}>
            <Field.Label>Mot de passe</Field.Label>
            <Input
              type="password"
              value={password}
              onChange={(e) => setPassword(e.target.value)}
            />
            {isPasswordError && (
              <Field.ErrorText>Le mot de passe est requis</Field.ErrorText>
            )}
          </Field.Root>

          <Button
            type="submit"
            width="full"
            mt={4}
            variant="outline"
            loading={isSubmitting}
          >
            Se connecter
          </Button>
        </VStack>
      </form>
    </Box>
  );
};

export default LoginPage;
```

</Example>

<Aside type="note" title="Points importants à comprendre">
- **État local** : React utilise `useState` pour gérer l'état local d'un composant. Chaque fois qu'un état change, le composant se re-render.
- **Événements** : les fonctions comme `onChange` et `onSubmit` sont des gestionnaires d'événements qui permettent de réagir aux actions de l'utilisateur.
- **Formulaires contrôlés** : en React, un formulaire est dit "contrôlé" lorsque ses valeurs sont stockées dans l'état et mises à jour via `onChange`.
- **Validation** : la validation peut être effectuée au moment de la soumission ou en temps réel. Pour des validations plus complexes, il existe des bibliothèques comme *React Hook Form*.
</Aside>
